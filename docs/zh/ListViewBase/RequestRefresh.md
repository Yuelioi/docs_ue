# RequestRefresh

设置列表在下一次打勾时刷新。

请注意，从列表的角度来看，刷新只限于说明项目和条目之间的差异。换句话说，它只会释放不再有项目的条目，并为新项目（或新的可见项目）生成条目。

它不考虑现有项目中的变化--那是由项目来宣布的，而且是根据需要来监听一个条目。对于简单的情况来说，这可能是一个繁琐的设置，所以当需要反映N个列表项中的变化时，调用RegenerateAllEntries也是合理的（尽管不是很理想）。
